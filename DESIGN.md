## Problem Statement

웹의 경제·주식·코인 데이터/뉴스를 수집해 AI 에이전트가 시황·이벤트를 해석하고,
"지금 영향이 클 종목·섹터"를 **근거와 함께** 제시하는 제품.

단, 이 프로젝트의 진짜 고객과 사용자가 확정되면서 제품의 성격이 바뀌었다.
이것은 리테일 투자자용 "종목 추천 앱"이 아니라, **중견 증권사 애널리스트
150명을 위한 리서치 가속 도구(B2B)** 다. 애널리스트가 본인 이름을 걸고 쓰는
보고서와 투자 참고자료의 **입력**을 빠르고 빠짐없이 만들어주는 것이 목표다.

핵심 파이프라인: 합법 소스 수집(크롤링/공시/뉴스 API) → dedup·클러스터링 →
엔티티/티커 링킹 → 이벤트·영향 분류 → **출처 추적 가능한** 영향도 분석 산출.

## Demand Evidence

- **이미 서명된 계약이 존재한다.** 견적 요청 후 계약 체결 완료.
- **사용 기업: 중견 증권사.** 실사용자 약 **150명**(전원 sell-side 애널리스트).
- **용도:** 애널리스트의 분석 보고서 작성 + 투자가 참고자료.
- 계약 납품물(원문): "주식/코인에 영향을 줄 만한 모든 정보를 모아 분석해서
  분석리포트 제공 — 어떤 종목/섹터가 어떻게 영향받을지, 주목해야 할 섹터 등."

이것은 설문·대기자·"관심"이 아니라 **돈이 오간 실제 수요**다. 이번 세션에서
나온 가장 단단한 신호.

## Status Quo (그리고 명시된 갭)

중견 증권사 애널리스트가 현재 쓰는 것으로 추정되는 도구(조사 기반):

- **연합인포맥스 단말** — 실시간 뉴스·공시·시세의 사실상 한국 표준(월 55만원/대).
- **FnGuide DataGuide / QuantiWise** — 재무·컨센서스·밸류에이션 엑셀 Add-in
  (계정당 연 720만원). 기계적 대량추출·DB구축·재배포 **계약상 금지**.
- **OpenDART(무료)·한경 컨센서스(무료)·인포스탁 테마 DB** — 공개/무료 소스.
- 일부 좌석에 한해 **Bloomberg**(좌석당 연 ~4,300만원, 중견사는 소수만 보유).

추정 일과: 07:00 전후 출근 → 섹터 뉴스 클리핑 → 08:00 모닝미팅 → 09:00 장 대응
→ 오후 리포트 작성. 실적시즌(2·3·5·8·11월)에 업무 폭증. 반복 업무(클리핑·데이터
취합·표/그래프)가 분석·집필 시간을 잠식.

> **⚠️ 명시된 갭 (가장 큰 리스크):** founder는 이 150명이 *실제로* 아침마다
> 무엇을 하는지 아직 모른다. status quo가 곧 진짜 경쟁자이므로, 이걸 모르면
> "무엇이 더 나은가"도 정의할 수 없다. → MVP 실사용으로 먼저 메워야 할 갭이고,
> 이것이 "The Assignment"의 핵심이다.

## Target User & Narrowest Wedge

- **사용자(매일 켤 사람):** 중견 증권사 sell-side 애널리스트. 본인 실명으로
  보고서를 쓰는 전문가. AI가 내린 결론을 *그대로* 신뢰하지 않으며, 환각(허위
  인용·틀린 수치) 한 건이면 도구를 영구히 외면한다.
- **구매자(서명한 사람) ≠ 사용자.** 계약은 임원/리서치 의사결정자가 "AI가 종목을
  찍어주는 리포트"라는 기대로 샀을 가능성이 크다. 계약대로 완벽히 납품해도
  애널리스트 150명이 안 켜면 갱신은 없다. 두 기대를 동시에 만족시켜야 한다.
- **가장 좁은 쐐기(wedge):** 시간압박이 최대인 **모닝미팅 직전 구간(07:00)**.
  장전에 "내 커버리지 종목·섹터에 영향 줄 뉴스·공시·이벤트"를 출처·발행시각과
  함께 5분 만에 보여주는 일간 **증거 브리프**. 여기서 한 번이라도 쓰게 만든다.

## Constraints

- **유니버스:** 한국 + 미국 주식, 메이저 코인(BTC, ETH, SOL, RWA 내러티브 토큰).
- **납품:** 주간 리포트 기본 + **온디맨드 트리거**(즉시 실행). **웹 대시보드** UI.
- **타임라인:** 유동적/넉넉함 → 빨리가 아니라 제대로 만들 시간이 있다.
- **도메인 규칙(CLAUDE.md):** 종목 "추천"은 투자 권유가 아니라 "현재 뉴스 기준
  영향도 분석". 추천 근거(어떤 뉴스·이벤트)와 소스 신뢰도·발행시각을 항상 병기.

## Premises (Phase 3에서 전부 동의됨)

- **P1.** 예측·추천이 아니라 **추적 가능한 영향도 분석 + 신뢰도 등급(HIGH/MED/LOW)**.
  근거: 금융 NLP에서 반복 확인된 결과 — 뉴스 감성→단기 주가 방향 예측은 다수클래스
  베이스라인을 안정적으로 못 넘는다(조사 인용: arXiv 2606.12210, 37.5% vs 48.4%,
  최고 43.75%; 신뢰도 HIGH 분류 건 49.4% 정확). ⚠️ **이 특정 인용 ID·수치는 착수
  전 원문 재확인 필요**(조사 단계 인용). 다만 "예측은 약하고, 영향도 해석·신뢰도
  등급이 유효하다"는 방향성 결론은 이 한 편에 의존하지 않는 robust한 결론.
- **P2.** 추적성이 "기능"이 아니라 **제품 본체**다. 결론은 얇게, 증거는 두껍게.
  모든 문장이 발행시각 찍힌 1차 소스로 클릭-투-점프 안 되면 전문가는 거부한다.
- **P3.** AI는 단독 저자 불가. **애널리스트가 검토·실명 확인 후 발간**하는 보조
  초안 생성기로 한정 → AI 공급사는 유사투자자문/투자자문 인허가 대상이 아닌
  솔루션 공급자로 남는다(추정). (KOFIA 영업·업무규정 제2-27조 실명책임). D2C로
  개인에게 직접 팔면 즉시 인허가 트리거. ⚠️ **이건 합리적 해석이나 법적 결론 —
  착수/계약 이행 전 자문 변호사 확인 필요(Open Q2와 연결).** 확정 전까지는 "동의된
  전제"가 아니라 열린 법무 항목으로 취급한다.
- **P4.** "모든 정보"는 범위 트랩. **정의된 합법 소스 + 정의된 유니버스**로 못박는다.
- **P5.** 한국 뉴스 본문 직접 크롤링 금지. 네이버 약관 + 저작권법 §93
  데이터베이스제작자권(잡코리아 vs 사람인 손배 4.5억). → 네이버 검색 오픈API
  (헤드라인+요약+URL) 또는 BIGKINDS/KPF 라이선스 경유. FnGuide·인포맥스·Bloomberg는
  스크래핑 금지 — 고객사 보유 좌석 읽기 연동만 합법.
- **P6.** 차별화는 "뉴스 수집"(이미 인포맥스/Bloomberg/DART가 가진 table stakes)이
  아니라 **합성·속도·KR+글로벌+코인 교차커버·추적성·이견 가시화·컴플라이언스 내장**.

## Approaches Considered

### Approach A: 모닝미팅 증거 브리프 (최소 실행 / Traceability-First MVP)
장전 07:00 모닝미팅용 증거 브리프. ("단방향"=사용자와 1:1 종목 자문을 주고받지
않는다는 규제적 의미이며, 대시보드 자체는 클릭-투-소스 인터랙티브다.) 합법 소스에서 유니버스 이벤트만 수집
→ dedup → 클러스터링 → 티커 링킹 → 이벤트 타입 분류. 결론 얇게(영향 종목 후보 +
이벤트 타입 + 신뢰도), 증거 두껍게(클릭→발행시각 찍힌 원문). 분석문은 Anthropic
Citations API로만 생성. 주간 리포트 = 일간 브리프 집계.
- Stack: 네이버 오픈API + OpenDART + SEC EDGAR + CoinGecko + KRX(EOD) / 고정
  파이프라인(멀티에이전트 없음) / pgvector / Citations API / 단일 VM + 07:00 크론.
- Effort: **M (≈2~3주)** · Risk: **Med** (네이버 본문 미제공 → 깊이 얕음, 2차 영향 없음).

### Approach B: Source-Traceable 리서치 OS (이상적 설계)
6레이어 파이프라인을 감사 추적 가능한 에이전트 오케스트레이션 위에. 차별화 ①
하이브리드 데이터(무료 1차 + 고객 보유 인포맥스/FnGuide 좌석 읽기연동 + BIGKINDS
본문 라이선스) ② DART 사업보고서 "주요 매출처/매입처" 추출로 경량 공급망 KG →
직접 언급 안 된 2차 수혜·피해주 ③ 이중 평가(RAGAS faithfulness CI + event study
CAR 백테스트). 컴플라이언스(실명확인·이해상충·AI고지·단정어 가드레일) 기능화.
- Stack: LangGraph + LangSmith / Citations API + 생성후 NLI entailment 검증(2-패스)
  / pgvector·Qdrant / KIS·LS 증권사 API(KR 실시간) / RWA.xyz·Dune / K8s·ECS.
- Effort: **XL (≈8~12주+)** · Risk: **Med-High** (라이선스 협상 외생 변수, 범위 큼).

### Approach C: 이견 레이더 + 증거 검색 코필럿 (창의적 전환)
"리포트 생성기"→"애널리스트 코필럿 + 추적 가능 증거 피드"로 재프레이밍. 축1
**이견 레이더**: 한경 컨센서스 다수 증권사 리포트 + FnGuide 컨센서스를 합성해
"의견 갈리는 지점·추정치 이상치·애널리스트 정확도 스코어" 하이라이트(매수 93%
편향 불신 역이용). 축2 **증거 검색 코필럿**: 자연어 질의 → Citations API가 원문
문장·공시 항목을 side-by-side로 추출, 결론 없이 근거 조각만. MCP/API로 노출 →
인포맥스 엑셀·ChatGPT 워크플로 *안으로*.
- Stack: 한경 PDF RAG + FnGuide(좌석 읽기연동) + OpenDART + 인포스탁 테마 DB +
  MCP 서버 / Citations API / pgvector / 경량 단일 서비스.
- Effort: **L (≈4~6주)** · Risk: **Med** (재프레이밍 가설 검증 필요, 한경 PDF 저작권 게이트).

## Recommended Approach

**단계적: A로 진입 → C의 이견 레이더로 차별화 → 검증 후 B로 확장.** (사용자 선택)

근거를 제약에 매핑:
1. 채택 결정권은 150명 애널리스트에게 있고, 이들을 한 번이라도 쓰게 만드는 게
   최우선 → 시간압박 최대인 모닝미팅을 잡는 **A(MVP)** 가 이번 달 가치 증명에 최선.
2. 단순 리포트 생성은 곧 commoditize(미래에셋 AI리서치센터 5h→15min·RA폐지, KB,
   한투 AIR, 딥서치, 토스증권) → **C의 이견 레이더**를 차별화 축으로 조기 병합.
3. 타임라인이 넉넉한 건 **B**를 정당화하나, *애널리스트 일과를 모르는 갭*이
   메워지기 전 XL부터 짓는 건 범위 트랩. A 실사용으로 워크플로를 배운 뒤 검증된
   부분만 B(공급망 KG·평가 이중화·라이선스 데이터 연동)로 확장.

**한 줄: A로 들어가서 C로 차별화하고 B로 키운다.**

> **Stage 1 단독 가치의 솔직한 한계 (리뷰 반영):** 무료 데이터만 쓰는 Stage 1은
> KR 뉴스가 헤드라인+요약 수준(본문 미수집, P5)이라 **KR 종목의 추적성은 헤드라인/
> 요약 단위**로 제한된다. 본문 단위 grounding은 BIGKINDS/KPF 라이선스(Stage 3)가
> 있어야 가능. 따라서 Stage 1이 인포맥스/Bloomberg 대비 *명백히 더 나은* 지점은
> "모든 뉴스 수집"이 아니라 ① KR+US+코인 한 화면 교차 + ② 이벤트→영향 종목 자동
> 링킹 + ③ 공시(OpenDART/EDGAR, **본문까지 합법 수집**) 기반 sentence-level 추적성이다.
> 이 셋만으로 status quo를 못 이기면 Stage 2로 가기 전 방향을 재검토한다(검증 게이트).

### 단계별 빌드 순서 (검증 게이트 포함)

- **Stage 0 (착수 전, 코드 아님):** 애널리스트 3~5명 워크플로 관찰/인터뷰
  → verify: "가장 큰 시간 누수 구간"을 1~2개로 특정. **병렬 가능:** 합법 소스
  커넥터·수집 인프라(아래 Stage 1 데이터 레이어)는 Stage 0과 동시 착수 가능 —
  UX·범위·"무엇을 브리프에 넣나" 결정만 Stage 0 완료에 막힌다.

- **Stage 1 (A, 2~3주) — 인터랙티브 증거 브리프 대시보드:**
  - 데이터: 네이버 검색 오픈API + OpenDART + SEC EDGAR + CoinGecko + KRX(**EOD로 확정**).
    - *구현 현황(2026-06-23): KRX·CoinGecko 시세 수집은 미구현(이연). CoinGecko는 CRYPTO
      별칭 시더로만 구현. 현 일일 수집은 뉴스/공시 6소스(rss·naver·opendart_docs·edgar_docs·marketaux·finnhub).*
  - 파이프라인: dedup(제목 SimHash → 임베딩 cosine) → 클러스터링 → 티커 링킹 → 이벤트 분류.
  - 생성: Citations API, **문장 단위(sentence-level) 인용**.
  - 신선도: 미국장 마감(~06:00 KST)~07:00 사이 야간 이벤트를 반드시 포함하는 수집
    컷오프 + 브리프에 "마지막 갱신 시각" 표기.
  - 커버리지: 애널리스트별 커버 종목/섹터 리스트 온보딩·관리(분석 대상 정의).
  - 빈 상태/장애: 소스 다운·쿼터 소진·해당 종목 무이벤트 시 명시적 degraded/empty UX
    (지어내지 않고 "근거 없음" 표기 = null-evidence refusal).
  - verify: 애널리스트가 모닝미팅에 실제 사용 + 모든 문장이 클릭→발행시각 찍힌
    원문으로 점프(sentence-level) + 환각 인용 0건(swap test).

- **Stage 2 (C의 "이견 레이더" 슬라이스만, +2~3주):** ⚠️ Approach C 전체(L, 4~6주)가
  아니라 **축1(이견 레이더)만** 떼어낸 부분집합이라 +2~3주. 축2(증거 검색 코필럿)·
  MCP 노출은 Stage 3로 미룬다. 내용: 한경 컨센서스/인포스탁 합성 + 이견·추정치 이상치
  하이라이트. **선결 게이트:** 한경 컨센서스 PDF의 RAG 활용 저작권·ToS 적법성
  (Open Q4) 확인 전엔 착수 금지.
  → verify: status quo가 못 주는 "이견 가시화"를 애널리스트가 가치로 인정.

- **Stage 3 (B 선별 확장 + C 잔여):** Stage 1~2에서 검증된 부분만 — 공급망 KG(2차
  영향), RAGAS faithfulness CI + event study CAR 백테스트, 고객 좌석/BIGKINDS 라이선스
  데이터 연동(→ KR 뉴스 본문 단위 추적성·KR 실시간 시세), 증거 검색 코필럿 +
  **MCP/API 노출**(Distribution의 B2B2C 채널 — MCP는 여기서 단일 관리).

## Success Criteria

각 지표에 활성 Stage를 표기. 임계치는 착수 후 베이스라인으로 보정할 **초기 목표**.

- **채택 (Stage 1, 가장 중요):** 모닝미팅 전 브리프 주간 활성 애널리스트 비율.
  초기 목표 ≥ 30%(150명 중 ~45명)에서 출발, 갱신 협상 전 ≥ 50%.
  (측정창: Stage 1 출시 후 4주 롤링 주간 활성 기준.)
- **인용 정확도 (Stage 1, 배포 게이트):** 두 개의 다른 바를 구분한다 —
  (a) **조작/존재하지 않는 인용 = 0건**(하드 게이트, 절대 0; swap test·null-evidence
  refusal을 CI 회귀 테스트로), (b) **claim이 인용을 과장·왜곡하지 않는 NLI 함의율
  ≥ 98%**(2% 미만 미달분은 자동 플래그→애널리스트 검토로 회수, 배포는 차단 안 함).
- **티커 링킹 정확도 (Stage 1):** 뉴스→영향 종목 매핑 정밀도(precision) ≥ 95%;
  신뢰도 임계 미달이면 종목을 단정하지 않고 "후보" 표기 또는 보류(오귀속이 추적성
  신뢰를 깬다).
- **생성 품질 (Stage 3):** RAGAS faithfulness ≥ 0.9 (CI 게이트).
- **영향분석 유효성 (Stage 3):** event study CAR 백테스트에서 식별 이벤트의 통계적
  유의성(p < 0.05).
- **컴플라이언스 (Stage 1~):** 산출물이 애널리스트 실명 검토 후 사내 준법 게이트를
  통과해 실제 발간 가능.

## Distribution Plan

- **형태:** 고객 증권사 대상 **B2B 웹 대시보드(SaaS)**. 사내망/SSO 연동 가능성 검토.
  추가 채널로 **MCP 서버/API** 노출(인포맥스 엑셀·ChatGPT/Claude 등 기존 워크플로
  안에서 "근거 검색" 호출 → "또 하나의 창" 도입 마찰 해소, B2B2C 확장 여지).
- **CI/CD:** 컨테이너 빌드 + 배포 파이프라인(초기에는 단일 VM/컨테이너 + 07:00
  크론, 확장 시 K8s/ECS 이벤트 드리븐 워커). RAGAS faithfulness·인용 회귀 테스트를
  배포 게이트로 포함.
- **데이터 운영:** OpenDART는 **캐싱·증분수집을 1차 수단**으로(공시는 불변이라 한 번
  받으면 재호출 불필요). 일 20,000건 한도 외에 **분/초 단위 throttling과 실적시즌
  대량 제출 스파이크**가 실제 병목 — 컷오프 일정·재시도 백오프 설계. 인증키 풀링은
  OpenDART ToS상 다중키 허용 여부 확인 전엔 의존하지 않는다(약관 리스크). 수집은
  별도 서비스로 분리.
