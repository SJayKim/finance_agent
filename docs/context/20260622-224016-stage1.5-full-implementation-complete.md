### Decisions Made

- **임베딩 dim 게이트(§6) 해소:** bge-m3·e5-large 둘 다 dim 1024라 `Vector(1024)` 고정 — greenfield(기존
  임베딩 0건)라 재임베딩 비용 없음. `embedding_model="BAAI/bge-m3"` 기본값, 임베더는 **pluggable**
  (sentence-transformers는 `[project.optional-dependencies] embeddings` 옵션 extra — 코어 uv sync 가벼움),
  미설정 시 graceful no-op. raw_documents.embedding + clusters.centroid → Vector(1024) + HNSW(vector_cosine_ops) 인덱스.
- **모델 미로드 보장:** run_pipeline의 embed 단계는 embedder 명시 주입 시에만 동작(자동 생성 안 함).
  /trigger·테스트는 embedder=None → no-op → 2GB 모델 절대 로드 안 됨. 일일 오케스트레이터(run_daily CLI/엔드포인트)만 get_embedder()로 실모델 주입.
- **RAG 경계 불변:** 누적 RAG 챗봇은 "무엇을 먹일지"(pgvector 코사인 검색으로 인용 span 선택)만 바꾸고
  "무엇을 신뢰할지"(인용 0 → None 거부)는 그대로. 한날치 챗봇(anthropic_chat) 그대로 보존, anthropic_rag_chat 추가.
- **다이제스트(트랙 C):** 그날 status=ok brief_items를 거시/섹터로 **집계**(자유 거시 서술 금지, D1). citations.py
  2-패스 경계 재사용, 인용 0/빈 날 → status=empty(무조작). daily_digests + digest_sources 역추적.
- **소스 격리(트랙 B):** run_daily에서 한 커넥터 장애(타임아웃·쿼터·키부재)가 나머지를 막지 않음 + audit_log 기록.
- **scope = 트랙 A~E만.** §8 추가 제품 아이디어(메신저/메일 푸시 등)는 외부 인프라 필요 → 이연(§8.6 소스헬스만 트랙 E에 반영).
- **API 키 부재 대응:** DART/SEC-UA/Marketaux/Finnhub 설정 필드는 있으나 값 없음 → 커넥터는 fixture 단위테스트
  (MockTransport)로 검증, 키 추가 시 즉시 라이브. naver 키만 config에 존재.
