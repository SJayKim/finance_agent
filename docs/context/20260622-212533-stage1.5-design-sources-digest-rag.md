## Working on: Stage 1.5 설계 — 소스 확장 · 일일 다이제스트 · 누적 RAG 챗봇

### Summary

office-hours(설계 세션)로 사용자 요청 5건(① 경제뉴스 소스 넓게 추가 ② 매일 크롤링→DB
저장 ③ LLM 일일 인사이트(거시·섹터) ④ 날짜별 요약 뷰 + 누적 코퍼스 챗봇 ⑤ 추가 제품
아이디어)을 기존 "증거 브리프" 파이프라인 위에 얹는 증분 설계로 정리했다. 산출물은
`docs/STAGE1.5_DESIGN.md`(DRAFT). **코드는 아직 안 짬 — 설계만.** 다음 세션은 트랙 B(일일
오케스트레이션)부터 구현하면 된다.

### Decisions Made

3개 핵심 포크를 사용자에게 물어 확정(설계 충돌 해소):
- **D1 = 기존 컴플라이언스 경계 유지(애널리스트 도구).** 요청 #3 "투자관련 인사이트"는
  자유 LLM 거시 전망이 아니라 **그날 brief_items를 거시·섹터 축으로 집계한 것**으로 정의.
  모든 문장이 인용으로 역추적. "투자권유 아님"·zero-fabrication 유지. 자유 거시 서술 금지.
- **D2 = 전체 코퍼스 의미검색(pgvector RAG).** 요청 #4 누적 챗봇은 임베딩 레이어(§11.3)를
  신설해 검색→인용 span을 Citations API에 먹임. **인용 0건이면 거부**(현재 web/chat.py와
  동일 경계). 검색은 "무엇을 먹일지"만 바꾸고 "무엇을 신뢰할지"는 안 바꿈.
- **D3 = 소스 4영역 전부 확장.** KR 주식·경제 뉴스 / 글로벌 매크로 / 공시 본문(DART·SEC) /
  코인 구조화 뉴스(Marketaux·Finnhub).

핵심 발견(난이도에 큰 영향):
- 실제 뉴스 수집기는 **코인 RSS 1개(app/collector/rss.py)뿐.** sec.py/opendart.py는 뉴스
  수집기가 아니라 **security_aliases(티커 사전) 시더.** KR 뉴스·글로벌 매크로·공시 본문
  수집은 전부 미구현.
- **일일 자동 수집이 통째로 없음.** /trigger(app/main.py)는 run_pipeline만 호출하고 수집기를
  안 부름. #2(매일 수집→저장)는 새 오케스트레이션(run_daily) + cron 필요.
- **임베딩 차원 미고정.** models.py의 embedding=Vector() 차원 없음, config의 embedding_model/
  dim = None. RAG는 모델 확정 + Vector(dim) 마이그레이션 + 인덱스가 비가역 선결 게이트.
