| 페인포인트               | 실제 업무상 문제                                                                        |
| ------------------------ | --------------------------------------------------------------------------------------- |
| 봐야 할 소스가 너무 많음 | 인포맥스, DART, 뉴스, 해외 뉴스, 블룸버그, FnGuide, 한경 컨센서스 등 창이 과도하게 많음 |
| 시간은 짧음              | 장 시작 전까지 핵심 이슈를 정리해야 함                                                  |
| 놓치면 치명적            | 커버리지 종목 관련 악재·호재를 놓치면 모닝미팅이나 고객 질의에서 바로 드러남           |
| 반복 작업 비중이 큼      | 클리핑, 복붙, 표 정리, 출처 확인에 시간이 많이 듦                                       |


즉, 애널리스트가 원하는 건 “AI가 종목을 찍어주는 것”이 아니라 **내 커버리지에 영향 줄 만한 것만 먼저 추려주는 필터**입니다.

---

### 2) “이 뉴스가 내 종목에 무슨 의미인가?”를 매번 수작업으로 판단해야 함

뉴스 자체는 이미 많습니다. 문제는 뉴스가 특정 종목·섹터·밸류체인에 어떤 영향을 주는지 연결하는 작업입니다.

예를 들면:

* “미국 금리 인하 기대 확대” → 은행, 증권, 성장주, 원달러 환율 영향
* “중국 희토류 수출 규제” → 2차전지, 반도체 장비, 방산 소재 영향
* “엔비디아 실적 서프라이즈” → 국내 반도체, HBM, PCB, 전력 인프라 영향
* “비트코인 ETF 자금 유입” → 코인거래소, 채굴, RWA, 관련 토큰 영향

애널리스트 입장에서는 단순 뉴스 요약보다 **이벤트 → 관련 종목/섹터 → 영향 방향 → 근거**가 필요합니다.

---

### 3) 리포트 작성 전 “입력 재료”를 모으는 시간이 너무 김

리포트의 핵심은 애널리스트의 판단이지만, 그 판단을 위해 필요한 원재료 수집이 매우 반복적입니다.

주요 반복 업무는 다음과 같습니다.

| 업무           | 페인                                                |
| -------------- | --------------------------------------------------- |
| 뉴스 클리핑    | 중복 기사 많고, 핵심만 뽑기 어려움                  |
| 공시 확인      | 수치·날짜·문구 확인에 시간 소모                   |
| 컨센서스 비교  | 추정치 변화, 목표가 변화, 의견 차이 파악이 번거로움 |
| 표/그래프 생성 | FnGuide, 엑셀, 기존 리포트 간 이동이 많음           |
| 근거 링크 정리 | 나중에 검증하려면 출처 추적이 필요함                |

그래서 이 제품의 강한 가치는 **완성 리포트 자동 생성**보다는 **리포트 작성용 재료 패키징**에 있습니다.

## 2. 리서치센터/증권사 조직 차원의 페인포인트

### 1) 애널리스트 생산성 압박

리서치센터는 제한된 인력으로 더 많은 종목, 더 많은 섹터, 더 빠른 업데이트를 커버해야 합니다.

특히 중견 증권사는 대형사 대비 다음 문제가 큽니다.

| 구분        | 중견 증권사의 부담                                           |
| ----------- | ------------------------------------------------------------ |
| 인력        | 대형사보다 RA/주니어 리소스가 부족할 수 있음                 |
| 데이터 비용 | Bloomberg, FnGuide, 인포맥스 좌석을 무제한으로 늘리기 어려움 |
| 커버리지    | 한국·미국·코인까지 커버 범위가 넓어짐                      |
| 고객 대응   | 기관/리테일/영업부서 질의에 빠르게 대응해야 함               |

즉, 조직 입장에서는 **애널리스트 1명당 처리량을 늘리고 싶다**는 페인이 있습니다.

---

### 2) 리서치 품질의 편차

애널리스트마다 정보 수집 방식, 소스, 정리 습관이 다릅니다. 어떤 사람은 공시를 먼저 보고, 어떤 사람은 뉴스 단말을 먼저 보고, 어떤 사람은 텔레그램/메신저/해외 뉴스부터 봅니다.

이러면 조직 입장에서 다음 문제가 생깁니다.

* 중요한 이슈를 누가 발견했는지에 따라 대응 속도 차이 발생
* 같은 이벤트에 대해 섹터별 해석이 따로 놀 수 있음
* 주니어와 시니어 간 리서치 입력물 품질 차이 큼
* 모닝미팅 자료의 포맷과 깊이가 들쭉날쭉함

따라서 증권사 입장에서는 **표준화된 증거 브리프**가 필요합니다.

---

### 3) 실적 시즌 업무 폭증

기획안에도 실적시즌인 2·3·5·8·11월에 업무 폭증이 언급되어 있습니다.

실적 시즌에는 다음 작업이 몰립니다.

* 실적 발표 확인
* 컨센서스 대비 서프라이즈/쇼크 계산
* 가이던스 변화 확인
* 경쟁사 실적과 비교
* 목표주가/추정치 조정
* 고객사 질의 대응
* 리포트 빠른 발간

이때 AI 리서치 도구가 줄여줄 수 있는 페인은 **실적 발표 후 1차 정리 시간**입니다.

예를 들면:

> “삼성전기 2Q 실적 발표 → 컨센서스 대비 매출 +3%, 영업이익 -5% → MLCC 마진 하락 원인 → 관련 공급망/동종사 영향 → 확인 가능한 공시·기사 링크”

이런 식으로 초안을 빠르게 만들어주면 실적 시즌에 체감 가치가 큽니다.

## 3. 신뢰성/컴플라이언스 페인포인트

### 1) AI 환각에 대한 강한 불신

증권사 리서치에서 가장 위험한 것은 **틀린 수치, 없는 출처, 과장된 해석**입니다.

애널리스트는 자기 이름으로 보고서를 냅니다. 따라서 AI가 다음 중 하나라도 하면 바로 신뢰를 잃습니다.

* 존재하지 않는 기사 인용
* 공시 수치 오기재
* 기사 내용을 과장해서 해석
* 특정 종목을 잘못 연결
* 날짜·시간 오류
* “확정적 표현”으로 투자 권유처럼 보이는 문장 생성

그래서 이 시장의 핵심 페인은 **AI가 답을 잘하느냐**가 아니라 **AI가 틀렸을 때 책임을 누가 지느냐**입니다.

---

### 2) 투자자문/유사투자자문 리스크

기획안에도 제품 방향이 “예측·추천”이 아니라 **추적 가능한 영향도 분석**이어야 한다는 전제가 있습니다.

증권사 입장에서는 다음 경계가 중요합니다.

| 위험한 표현              | 안전한 방향                          |
| ------------------------ | ------------------------------------ |
| “이 종목 매수 추천”    | “현재 뉴스 기준 영향 가능성 있음”  |
| “상승할 가능성이 높음” | “이벤트 성격상 긍정 요인으로 분류” |
| “목표가 상향 필요”     | “컨센서스 변화 여부 확인 필요”     |
| “투자 유망 섹터”       | “주목할 섹터 후보”                 |

즉, 증권사는 AI 도구가 **투자 판단을 대신하는 것처럼 보이는 것**을 매우 부담스러워할 수 있습니다.

---

### 3) 출처 추적 불가능한 요약은 실무에서 못 씀

애널리스트는 요약만 보고 움직이지 않습니다. 반드시 원문 확인이 필요합니다.

따라서 페인은 다음입니다.

* AI 요약은 그럴듯한데 원문 어디에서 나온 말인지 모름
* 기사 제목만 있고 실제 본문 근거가 약함
* 여러 기사 내용을 섞어서 결론을 냈는데 어떤 문장이 어느 출처인지 모름
* 발행 시각이 불명확해서 장중/장전 판단이 어려움

그래서 이 제품의 핵심 기능은 **요약 생성**이 아니라 **문장 단위 출처 추적**이어야 합니다.

## 4. 데이터/라이선스 페인포인트

### 1) 이미 쓰는 단말과 데이터 서비스가 강함

증권사는 이미 다음 도구를 쓰고 있습니다.

* 연합인포맥스
* FnGuide DataGuide / QuantiWise
* OpenDART
* 한경 컨센서스
* Bloomberg 일부 좌석
* 인포스탁 테마 DB

기획안에서도 이 status quo를 가장 큰 경쟁자로 보고 있습니다.

따라서 증권사의 페인은 “데이터가 없다”가 아닙니다. 오히려 **데이터는 있는데 연결·정리·해석이 분절되어 있다**가 정확합니다.
